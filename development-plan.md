# 基于根目录 PDF 的开发计划

## 1. 背景说明

根据根目录 PDF《作物建模与应用系统开发（上）》中的要求，本项目后续开发应围绕“玉米模型功能界面系统”展开，并完成三级界面设计与主要业务模块落地。

结合当前项目现状：

- 已具备 Electron + React + TypeScript + SQLite 基础架构
- 已完成部分主干模块，如地块、品种、种植记录、知识库、决策、设置等
- 已形成 `main / renderer / shared` 三层结构

因此，新的开发重点应从“基础框架搭建”转向“按 PDF 要求完成结构化补全与界面对齐”。

---

## 3. 当前项目与 PDF 的对应关系

### 3.1 已具备基础的部分

当前项目中已存在以下页面或能力：

- `home`：可作为主界面 / 系统介绍基础
- `fields`：可作为数据管理支撑模块
- `crop-varieties`：可作为参数与品种管理支撑模块
- `planting-records`：可作为试验数据管理基础
- `knowledge-items`：可作为专家知识库基础页
- `decisions`：可扩展为常年决策 / 当年决策入口
- `settings`：可承接参数调整 / 系统帮助部分功能

### 3.2 当前主要差距

- 页面命名与 PDF 二级模块未完全对齐
- 二级页面数量不足，尚未完整覆盖 10 个模块
- 三级页面中的模拟、预测、参数调整功能尚未完整落地
- 土壤数据管理、病虫害知识库、系统帮助等页面仍需补齐
- 整体视觉风格还需要统一为庄重、规范的农业管理系统风格

---

## 4. 总体开发目标

在现有项目架构基础上，完成以下目标：

1. 建立与 PDF 一致的三级界面结构
2. 补齐全部二级功能模块
3. 落地 4 个指定三级功能页面
4. 打通主要数据链路（数据库、IPC、前端页面）
5. 统一界面视觉风格，满足课程展示要求

## 阶段三：三级页面功能落地

### 目标
完成 PDF 指定的 4 个三级界面。

### 功能归属建议

#### 1. 生育期试验数据管理
归属：试验数据管理

建议内容：
- 试验批次列表
- 生育期记录录入
- 生育阶段查询与编辑
- 数据明细查看

#### 2. 适宜生育进程模拟
归属：常年决策

建议内容：
- 品种、地区、土壤条件输入
- 生育进程模拟触发
- 模拟结果表格 / 阶段展示
- 结果摘要与建议

#### 3. 生育进程预测
归属：当年决策

建议内容：
- 当前种植记录选择
- 关键阶段预测
- 预测结果展示
- 预测时间节点说明

#### 4. 生育期模型参数调整
归属：参数调整

建议内容：
- 参数分组展示
- 默认值 / 当前值对照
- 调整、重置、保存
- 参数修改记录

### 阶段输出

- 4 个三级页面可访问
- 三级页面与对应二级页面形成层级关系
- 演示路径可走通

## 阶段五：界面优化与交付准备

### 目标
满足 PDF 对展示效果的要求。

### UI 优化方向

- 主色调：深绿、墨绿、深蓝灰
- 辅助色：浅灰、米白
- 视觉风格：庄重、规范、简洁
- 页面布局：分区明确、标题清晰、信息密度适中

### 重点优化项

- 左侧导航样式统一
- 页面标题区统一
- 按钮、表单、表格风格统一
- 卡片、空状态、提示语统一
- 农业主题相关图标适度加入

### 交付前检查项

- 菜单跳转完整性检查
- 页面层级检查
- 假数据与演示数据准备
- 功能演示路径梳理
- 桌面端打包运行检查

### 阶段输出

- 界面风格统一版本
- 可演示版本
- 可提交版本
